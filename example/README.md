# Example — Marginalia

A worked example of an Eidos product definition. **Marginalia** is a small,
fictional SaaS: a shared reading library where teams save, annotate, and discuss
articles. This folder shows what a filled-in registry looks like end to end.

Copy this folder into your own repo as `product/` and replace the contents with
your product. Nothing points at the folder name by path, so rename it freely.

```
example/
  Architecture.md
  Audience.md
  Criteria.md
  Market.md
  domains.md
  specs/
    identity/
      magic-link-signin.md
      session-management.md
    library/
      saved-articles.md
```
